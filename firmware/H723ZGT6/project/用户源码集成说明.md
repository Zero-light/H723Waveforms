# Firmware Integration Guide

These source files (`app_*.c/h`) are designed to be dropped into a CubeMX-generated STM32H723 project.

## CubeMX Configuration

### 1. Clock
- Set SYSCLK to **550 MHz** (H723 max)
- APB1 timer clock will be derived accordingly (usually 275 MHz; timers see 2×PCLK1 if prescaler > 1)

### 2. USB
- **USB_OTG_FS** → Mode: **Device Only**
- In **Middleware > USB_DEVICE**, select **Communication Device Class (CDC)**
- This generates `usbd_cdc_if.c/h`

### 3. TIM2 (Waveform update timer)
- **TIM2** → Clock Source: **Internal Clock**
- **No Channel** needed (we use Update event only)
- NVIC: enable **TIM2 global interrupt** (optional, for debug/future use)

### 4. DMA1 (Waveform DMA)
- Add **DMA Request: TIM2_UP**
- Direction: **Memory to Peripheral**
- Mode: **Circular** (so pattern loops forever)
- Peripheral: `GPIOA->BSRR` (we will set this in code, not CubeMX)
- Data width: **Word** (32-bit) for both memory and peripheral

### 5. GPIO
- **PA0, PA1, PA2, PA3, PA5** → GPIO_Output, Initial State Low, No Pull-up/Pull-down, Speed Very High
- Keep PA4 free (DAC1_OUT1 reserved for analog expansion)

### 6. Project
- Code Generator: copy only necessary library files
- Generate code

## File Placement

Copy into your project:

```
Core/Inc/app_protocol.h
Core/Inc/app_wavegen.h
Core/Src/app_protocol.c
Core/Src/app_wavegen.c
```

## Modify Generated Files

### `USB_DEVICE/App/usbd_cdc_if.c`

At the top, add:
```c
#include "app_protocol.h"
```

In `CDC_Receive_FS()`, forward every byte to the protocol parser:

```c
static int8_t CDC_Receive_FS(uint8_t* Buf, uint32_t *Len)
{
    for (uint32_t i = 0; i < *Len; i++) {
        APP_Protocol_ParseByte(Buf[i]);
    }
    USBD_CDC_SetRxBuffer(&hUsbDeviceFS, &Buf[0]);
    USBD_CDC_ReceivePacket(&hUsbDeviceFS);
    return (USBD_OK);
}
```

### `Core/Src/main.c`

At the top, add:
```c
#include "app_protocol.h"
#include "app_wavegen.h"
```

Add a frame handler callback (before `main()`):

```c
static void OnFrame(const Frame_t *frame)
{
    switch (frame->cmd) {
        case CMD_WAVE_CONFIG: {
            if (frame->len >= 7) {
                WaveConfig_t cfg;
                cfg.sample_rate_hz = (frame->payload[0])
                                   | (frame->payload[1] << 8)
                                   | (frame->payload[2] << 16)
                                   | (frame->payload[3] << 24);
                cfg.num_points     = (frame->payload[4]) | (frame->payload[5] << 8);
                cfg.ch_mask        = frame->payload[6];
                bool ok = APP_WaveGen_Configure(&cfg);
                APP_Protocol_SendAck(CMD_WAVE_CONFIG, ok);
            }
            break;
        }
        case CMD_WAVE_DATA: {
            uint16_t n = frame->len / 4;
            bool ok = APP_WaveGen_LoadData((const uint32_t *)frame->payload, n);
            APP_Protocol_SendAck(CMD_WAVE_DATA, ok);
            break;
        }
        case CMD_WAVE_CTRL: {
            bool start = (frame->len > 0 && frame->payload[0] != 0);
            bool ok = start ? APP_WaveGen_Start() : (APP_WaveGen_Stop(), true);
            APP_Protocol_SendAck(CMD_WAVE_CTRL, ok);
            break;
        }
        default:
            APP_Protocol_SendAck(frame->cmd, false);
            break;
    }
}
```

In `main()`, before the main loop:
```c
APP_Protocol_Init(OnFrame);
APP_WaveGen_Init();
```

## DMA Handle Names

`app_wavegen.c` expects these external handles generated by CubeMX:
- `TIM_HandleTypeDef htim2;`
- `DMA_HandleTypeDef hdma_tim2_up;`

If your CubeMX uses different names (e.g., `hdma_tim2_update`), update `app_wavegen.c` accordingly.

## Build & Flash

1. Build in Keil / STM32CubeIDE / VS Code
2. Flash via STM32CubeProgrammer (USB DFU or SWD)
3. Connect USB Type-C to PC — it should enumerate as a Virtual COM port
4. Run the Python host and connect to the COM port
